<h2 id="introduction-to-xpath-and-contains">Introduction to XPath and&#xa0;<code>contains()</code></h2><img alt="XPath contains: How to select elements by text" src="https://storage.ghost.io/c/f2/6e/f26ec999-9a90-4aee-a0d4-9b3ca2bb668f/content/images/2024/03/XPath-contains-text_-how-to-select-elements-by-text.png" /><p>XPath stands for XML Path Language. It&apos;s a language used for navigating through elements and attributes in an XML document. Given its flexibility and precision, XPath is widely used in <a href="https://blog.apify.com/what-is-web-scraping/" rel="noreferrer">web scraping</a> and automation testing to search through the HTML structure of web pages. </p><p>XPath&apos;s utility can handle dynamic content, which is where the&#xa0;<code>contains()</code>&#xa0;method comes in handy. The&#xa0;<code>contains()</code>&#xa0;function is great at matching elements that include a specified string within their text or attributes, making it a useful tool for dealing with web pages where content or attributes might change dynamically.</p><h2 id="xpath-syntax-how-it-works">XPath syntax: how it works</h2><p><a href="https://www.lambdatest.com/blog/complete-guide-for-using-xpath-in-selenium-with-examples/" rel="noreferrer">XPath</a> expressions can be categorized into absolute and relative paths. </p><p>Absolute XPath paths start from the root node and traverse down to the desired element, specifying each ancestor element in the hierarchy (e.g.,&#xa0;<code>html/body/div[0]/input</code>). However, this approach is not the best, as any change in the document structure can invalidate the path. </p><p>Relative XPath (e.g.,&#xa0;<code>//input[@type=&apos;text&apos;]</code>), on the other hand, allows for more flexibility and robustness, particularly when combined with functions like&#xa0;<code>contains()</code>. This method doesn&apos;t require a detailed path from the root, making it more resilient to changes in the document.</p><p>To further explain the process of creating selectors, consider using the browser&apos;s developer tools. Visit the&#xa0;<a href="https://www.google.com/">Google Search</a>&#xa0;page, right-click on any element, and then click on inspect. This will open the developer tools and highlight the selected element. Right-click on the element code and select&#xa0;<code>Copy</code>&#xa0;&#x2192;&#xa0;<code>Copy XPath</code>&#xa0;or&#xa0;<code>Copy full XPath,</code>allowing you to obtain an element&apos;s XPath. Comparing this with the&#xa0;<code>Copy selector</code>&#xa0;option, which provides a CSS selector, can help you understand the differences between using XPath and CSS selectors. This approach in the browser is not only a convenient way to generate selectors but also aids in building the logic for more complex use cases.</p><figure class="kg-card kg-image-card kg-card-hascaption"><img alt="XPath contains: How to select elements by text" class="kg-image" height="375" src="https://storage.ghost.io/c/f2/6e/f26ec999-9a90-4aee-a0d4-9b3ca2bb668f/content/images/2024/03/Xpath-Syntax-Anatomy.-Copy-selector--Copy-XPath--Copy-full-XPath-1.png" width="1439" /><figcaption><span style="white-space: pre-wrap;">XPath syntax anatomy</span></figcaption></figure><ul><li><code>//tagname[contains(@attribute, &apos;value&apos;)]//</code> represents a relative path to the current page. </li><li><code>node tagname</code> represents a reference to an element, e.g., input, div. </li><li><code>@</code> is used to select the attribute of an element. e.g., id, class, name.</li><li><code>value</code> is used to reference the value contained in the attribute selected.</li></ul><h2 id="using-contains-to-select-elements-by-text">Using&#xa0;<code>contains()</code>&#xa0;to select elements by text</h2><p>The&#xa0;<code>contains()</code>&#xa0;function can significantly simplify the task of selecting elements that include specific text. The syntax for using&#xa0;<code>contains()</code>&#xa0;is as follows:&#xa0;</p><p><code>//div[contains(text(), &apos;apify&apos;)]</code>. This expression will select any&#xa0;<code>div</code>&#xa0;element containing&#xa0;<code>apify</code>. </p><p>The strength of&#xa0;<code>contains()</code>&#xa0;lies in its ability to handle elements with dynamic text or attributes. This enables users to target content that may not be exactly predictable.</p><h3 id="using-xpath-contains-and-advanced-xpath-techniques">Using XPath contains and advanced XPath techniques</h3><p>To showcase the practical applications of&#xa0;<code>contains()</code>, consider the following examples.</p><hr /><ul><li><strong>Example 1</strong></li></ul><p>Imagine you have an HTML document representing a blog post, which includes various sections, comments, and nested replies. Your goal is to find a specific comment by a user named &quot;John Doe&quot; and retrieve all replies to that comment.</p><pre><code class="language-html">&lt;div class=&quot;post&quot;&gt;
    &lt;h2&gt;Simple Blog Post&lt;/h2&gt;
    &lt;div class=&quot;comment&quot; id=&quot;comment1&quot;&gt;
        &lt;p class=&quot;author&quot;&gt;Jane Smith&lt;/p&gt;
        &lt;div class=&quot;body&quot;&gt;This is a comment.&lt;/div&gt;
    &lt;/div&gt;
    &lt;div class=&quot;comment&quot; id=&quot;comment2&quot;&gt;
        &lt;p class=&quot;author&quot;&gt;John Doe&lt;/p&gt;
        &lt;div class=&quot;body&quot;&gt;I have a question.&lt;/div&gt;
        &lt;div class=&quot;reply&quot;&gt;
            &lt;p class=&quot;author&quot;&gt;Alice&lt;/p&gt;
            &lt;div class=&quot;body&quot;&gt;Here&apos;s an answer.&lt;/div&gt;
        &lt;/div&gt;
        &lt;div class=&quot;reply&quot;&gt;
            &lt;p class=&quot;author&quot;&gt;Bob&lt;/p&gt;
            &lt;div class=&quot;body&quot;&gt;Another perspective.&lt;/div&gt;
        &lt;/div&gt;
    &lt;/div&gt;
&lt;/div&gt;

</code></pre><div class="kg-card kg-callout-card kg-callout-card-blue"><div class="kg-callout-emoji">&#x1f4a1;</div><div class="kg-callout-text"><b><strong style="white-space: pre-wrap;">Goal</strong></b>: Find and select all replies to John Doe&apos;s comment.</div></div><pre><code class="language-xml">//div[contains(@class, &apos;comment&apos;) and descendant::p[contains(@class, &apos;author&apos;) and contains(text(), &apos;John Doe&apos;)]]/div[contains(@class, &apos;reply&apos;)]

</code></pre><ul><li><strong><code>//div[contains(@class, &apos;comment&apos;)]</code></strong>: <br />Selects all divs that have a class containing &quot;comment&quot;.</li><li><strong><code>descendant::p[contains(@class, &apos;author&apos;)</code></strong> and <code><strong>contains(text(), &apos;John Doe&apos;)]</strong></code>:<br />From those comment divs, it further narrows down to those having a descendant paragraph <code>(&lt;p&gt;)</code> with a class of &quot;author&quot; and containing the text &quot;John Doe&quot;.&#xa0;</li><li><strong><code>/div[contains(@class, &apos;reply&apos;)]</code></strong>: <br />Finally, it selects all direct child divs of the filtered comments that have a class containing &quot;reply&quot;, which are the replies to John Doe&apos;s comment.</li></ul><hr /><ul><li><strong>Example 2</strong></li></ul><p>Suppose you&apos;re working with a dynamic web page that includes a list of notifications. These notifications have varying content, and you want to select all notifications that contain the word &quot;urgent&quot;.</p><pre><code class="language-html">&lt;ul class=&quot;notifications&quot;&gt;
    &lt;li class=&quot;notification&quot; data-type=&quot;update&quot;&gt;System update available.&lt;/li&gt;
    &lt;li class=&quot;notification&quot; data-type=&quot;alert&quot;&gt;Urgent: Security alert!&lt;/li&gt;
    &lt;li class=&quot;notification&quot; data-type=&quot;message&quot;&gt;You have new messages.&lt;/li&gt;
    &lt;li class=&quot;notification&quot; data-type=&quot;alert urgent&quot;&gt;Urgent: Meeting at 3 PM.&lt;/li&gt;
&lt;/ul&gt;

</code></pre><div class="kg-card kg-callout-card kg-callout-card-blue"><div class="kg-callout-emoji">&#x1f4a1;</div><div class="kg-callout-text"><b><strong style="white-space: pre-wrap;">Goal</strong></b>: Select all notifications that contain the word &quot;urgent&quot;.</div></div><pre><code class="language-xml">//li[contains(@class, &apos;notification&apos;) and contains(@data-type, &apos;urgent&apos;)]

</code></pre><ul><li><code><strong>//li[contains(@class, &apos;notification&apos;)]</strong></code>:<br />Selects all <code>&lt;li&gt;</code> elements that have a class containing&#xa0;&quot;notification&quot;.</li><li><code><strong>contains(@data-type, &apos;urgent&apos;)</strong></code>: <br />Further filters those <code>&lt;li&gt;</code> elements to only those with a data-type attribute containing the word&#xa0;&quot;urgent&quot;.</li></ul><hr /><ul><li><strong>Example 3</strong></li></ul><p>Imagine you are working with an online bookstore webpage that lists books in a structured format. Each book is contained within a&#xa0;<code>div</code>&#xa0;with the class,&#xa0;<code>book</code>, and each&#xa0;<code>div</code>&#xa0;includes details like the book title, author, and a list of genres it belongs to. There&apos;s also an&#xa0;<a href="https://www.convertcart.com/blog/add-to-cart-conversion-rate" rel="noreferrer"><code>Add to Cart</code></a>&#xa0;button for each book.</p><pre><code class="language-html">&lt;div class=&quot;book&quot;&gt;
    &lt;h2&gt;Book Title A&lt;/h2&gt;
    &lt;p&gt;Author: Leom Ayodele&lt;/p&gt;
    &lt;ul class=&quot;genres&quot;&gt;
        &lt;li&gt;Fiction&lt;/li&gt;
        &lt;li&gt;Adventure&lt;/li&gt;
    &lt;/ul&gt;
    &lt;button&gt;Add to Cart&lt;/button&gt;
&lt;/div&gt;
&lt;div class=&quot;book&quot;&gt;
    &lt;h2&gt;Book Title B&lt;/h2&gt;
    &lt;p&gt;Author: Leom Ayodele&lt;/p&gt;
    &lt;ul class=&quot;genres&quot;&gt;
        &lt;li&gt;Non-Fiction&lt;/li&gt;
        &lt;li&gt;Science&lt;/li&gt;
    &lt;/ul&gt;
    &lt;button&gt;Add to Cart&lt;/button&gt;
&lt;/div&gt;

</code></pre><p><strong>:bulb: </strong></p><div class="kg-card kg-callout-card kg-callout-card-blue"><div class="kg-callout-emoji">&#x1f4a1;</div><div class="kg-callout-text"><b><strong style="white-space: pre-wrap;">Goal</strong></b>: Find the &quot;Add to Cart&quot; button for the book by &quot;Leom Ayodele&quot; that is in the &quot;Fiction&quot; genre.</div></div><pre><code class="language-xml">//div[contains(@class, &apos;book&apos;) and descendant::p[contains(text(), &apos;Leom Ayodele&apos;)] and descendant::li[contains(text(), &apos;Fiction&apos;)]]/button

</code></pre><ul><li><strong><code>//div[contains(@class, &apos;book&apos;)]</code></strong>:&#xa0;<br />Selects all div elements with a class that contains &quot;book&quot;.</li><li><code><strong>descendant::p[contains(text(), &apos;Leom Ayodele&apos;)]</strong>:</code><br />Further filters those div elements to only include those that have a descendant <code>&lt;p&gt;</code> element containing the text &quot;Leom Ayodele&quot;.</li><li><code><strong>descendant::li[contains(text(), &apos;Fiction&apos;)]</strong></code>:&#xa0;<br />Narrows down the selection to those div elements that also have a descendant <code>&lt;li&gt;</code> containing the text &quot;Fiction&quot;.&#xa0;</li><li><code><strong>/button</strong></code>:<br />Selects the child <code>&lt;button&gt;</code> element of the filtered divs, which is the&#xa0;&quot;Add to Cart&quot;&#xa0;button for the book that matches both the author and genre criteria.</li></ul><hr /><p><em>These examples showcase how to precisely target elements by navigating the DOM structure using XPath axes and logical operators. The descendant is used to look down the DOM tree from the selected div elements to find matching criteria deep within its structure, demonstrating a powerful way to locate elements that might not be directly accessible due to complex HTML structures.</em></p><hr /><p>When XPath cannot be used to find an element, either due to the page complexity or structure or dynamic content that changes the DOM after page load or any other reasons that may occur, below are useful approaches to handle such situations:</p><ul><li>CSS selectors are often simpler and can sometimes access elements more easily than XPath, especially for web pages with well-defined class and ID attributes. Modern web automation tools and libraries, like Selenium or Puppeteer, support CSS selectors alongside XPath, providing a good alternative for element selection.</li><li>In cases where elements are not immediately available due to asynchronous operations (e.g., AJAX calls that fetch data after the initial page load), using explicit waits or polling mechanisms can help. Libraries like Selenium provide explicit wait functions to wait for an element to become present or visible before attempting to interact with it.</li><li>Some automation tools offer functions to search for elements based on their visible text, not just their attributes. This can be an alternative when the structure around the text is too complex for a straightforward XPath, but the text content itself is predictable.</li><li>Sometimes, the limitations are not with XPath itself but with how a particular tool implements or supports XPath. Exploring different automation tools or libraries might reveal one that works better with the web page in question or offers more powerful features for dealing with dynamic content.</li></ul><p>Choosing the right approach depends on the specific challenges you&apos;re facing, such as the complexity of the web page, the nature of the <a href="https://blog.apify.com/what-is-a-dynamic-page/" rel="noreferrer">dynamic content</a>, the tools and libraries you&apos;re using, and the stability of the elements you&apos;re trying to interact with. In many cases, a combination of these strategies will provide the best results.</p><h2 id="xpath-tools-and-frameworks">XPath tools and frameworks</h2><p>Several tools and frameworks enhance the experience of using XPath, e.g., <a href="https://blog.apify.com/scrapy-vs-selenium/" rel="noreferrer">Selenium and Scrapy</a>. These environments provide robust support for XPath, including the&#xa0;<code>contains()</code>&#xa0;function, facilitating complex web scraping and automation tasks.</p><p>You can also test XPath expressions, including those with&#xa0;<code>contains()</code>&#xa0;by using the&#xa0;<code>$x()</code>&#xa0;function in your browser&apos;s console. For instance, on a&#xa0;<a href="https://www.google.com/">Google search page</a>, open the developer tools, click on the&#xa0;<code>console</code>&#xa0;tab, and input&#xa0;<code>$x(&apos;/html/body/div[1]/div[2]/div/img&apos;)</code>&#xa0;to select the Google image on the page. This method allows for immediate experimentation with XPath queries in your web browser, reducing the development process for your automation tasks.</p><p>Implementing the contains() function in both Selenium and Scrapy allows for flexible text matching within element attributes or text content, making it easier to locate elements that match specific criteria. Here are brief examples for each:</p><h3 id="selenium-with-javascript">Selenium with Javascript</h3><p>In Selenium, the <code>contains()</code> function is part of the XPath query. You can use it to find elements that contain specific text or attribute values. Here&apos;s how you might use it in a Selenium script written in JavaScript:</p><pre><code class="language-jsx">import { Builder, By } from &apos;selenium-webdriver&apos;;

// Initialize the WebDriver for Chrome
let driver = await new Builder().forBrowser(&apos;chrome&apos;).build();

try {
    // Open a webpage
    await driver.get(&apos;&lt;https://blog.apify.com/author/ayodele/&gt;&apos;);

    // Use contains() to find an element with a specific text
    let element = await driver.findElement(By.xpath(&quot;//a[contains(text(), &apos;upload files in Puppeteer&apos;)]&quot;));

    // Use contains() to find an element with a specific attribute value
    let elementWithAttribute = await driver.findElement(By.xpath(&quot;//div[contains(@class, &apos;post-info-&apos;)]&quot;));

    // Add your interaction with element here
    // For example, click the element: await element.click();

} finally {
    // Don&apos;t forget to close the driver
    await driver.quit();
}

</code></pre><p>This example demonstrates how to locate elements by partial text match and partial attribute value match using <code>contains()</code> in your XPath expressions within a Selenium script.</p><h3 id="scrapy-with-python">Scrapy with Python</h3><p><a href="https://blog.apify.com/web-scraping-with-scrapy/" rel="noreferrer">Scrapy is a powerful web scraping framework</a> that uses selectors to extract data from web pages. You can use the&#xa0;<code>contains()</code>&#xa0;function in Scrapy selectors to select elements that contain certain text or have attributes that contain a specified value.</p><p>Here&apos;s a basic example of using&#xa0;<code>contains()</code>&#xa0;in a Scrapy script:</p><pre><code class="language-python">import scrapy

class ExampleSpider(scrapy.Spider):
    name = &apos;example&apos;
    start_urls = [&apos;&lt;https://blog.apify.com/author/ayodele/&gt;&apos;]

    def parse(self, response):
        # Use contains() to select links that include the word &apos;Puppeteer&apos;
        tag_links = response.xpath(&quot;//a[contains(text(), &apos;puppeteer&apos;)]/@href&quot;).extract()

        # Log or further process the extracted data
        self.log(f&quot;Puppeteer Tag Links: {tag_links}&quot;)

</code></pre><p>This&#xa0;<code>Scrapy</code>&#xa0;example shows how to use&#xa0;<code>contains()</code>&#xa0;in XPath queries to extract links containing the text &quot;Puppeteer&quot;.</p><p>Both examples illustrate the basic usage of&#xa0;<code>contains()</code>&#xa0;in XPath expressions within the context of Selenium and Scrapy. These snippets provide a basic usage for locating elements based on partial matches, which is especially useful in dynamic web environments where exact matches are not always possible.</p><h2 id="using-xpath-in-web-scraping-and-testing">Using XPath in web scraping and testing</h2><p>XPath&apos;s precision makes it a preferred choice for web scraping and automation testing. However, it&apos;s important to consider other methods like CSS selectors, which might offer simpler solutions for certain tasks. The choice between XPath and other techniques depends on the specific requirements of the project, including the complexity of the web structures involved and the need for dynamic content handling.</p><h3 id="why-xpath-is-useful-for-web-scraping-and-automation-testing">Why XPath is useful for web scraping and automation testing</h3><ul><li><strong>Precision in selection</strong>: XPath allows for the precise selection of elements, even in complex and deeply nested HTML structures. This precision is invaluable when you need to extract specific data points from a webpage or interact with particular elements during automation testing.</li><li><strong>Dynamic content handling</strong>: XPath expressions can be designed to handle dynamic content. Functions like <code>contains()</code>, <code>starts-with()</code>, and <code>text()</code> enable the selection of elements based on partial matches, which is useful for dealing with elements that have dynamically generated attributes, such as the class names in React applications. These are not readable and are regenerated with each component or app update. This approach allows you to select elements in a way that is resilient to changes in the DOM structure or attribute values.</li><li><strong>Traversal flexibility</strong>: XPath supports navigating the HTML DOM with various axes such as ancestor, descendant, following-sibling, and preceding-sibling. This flexibility allows users to select elements based on their relationships to other elements, which is particularly useful in complex web pages where direct selection is not straightforward.</li><li><strong>Logical operators</strong>: XPath expressions can use logical operators (and, or) to combine multiple conditions. This capability enables fine-tuned element selection based on multiple criteria, enhancing the effectiveness of both web scraping and automation testing tasks.</li><li><strong>Support across tools</strong>: Most web scraping and automation testing tools and libraries (like Scrapy, Selenium, and others) offer built-in support for XPath, making it a universally applicable skill set across different technologies.</li></ul><h3 id="alternatives-to-xpath-for-specific-purposes">Alternatives to XPath for specific purposes</h3><p>While XPath is powerful, there are scenarios where other methods might be more efficient or better suited:</p><ul><li><strong>CSS selectors</strong>: When the goal is to select elements based on their class, id, or style attributes, CSS selectors can be more straightforward and readable than XPath.&#xa0;</li></ul><div class="kg-card kg-callout-card kg-callout-card-blue"><div class="kg-callout-emoji">&#x1f44d;</div><div class="kg-callout-text"><b><strong style="white-space: pre-wrap;">Advantage</strong></b>: CSS selectors are widely used in web development, making them more familiar to a broader audience. They can also be shorter and more succinct for simple selections based on class or ID.</div></div><ul><li><strong>JavaScript and browser APIs</strong>: In dynamic web applications where elements are loaded asynchronously or manipulated by JavaScript, using JavaScript in conjunction with browser APIs (like <code>document.querySelector()</code> and <code>document.querySelectorAll())</code> can directly interact with the DOM post-render.&#xa0;</li></ul><div class="kg-card kg-callout-card kg-callout-card-blue"><div class="kg-callout-emoji">&#x1f44d;</div><div class="kg-callout-text"><b><strong style="white-space: pre-wrap;">Advantage</strong></b>: This approach allows direct execution of JavaScript in the browser context, enabling real-time interaction with and manipulation of the DOM, which is beneficial for extracting data from heavily dynamic pages or for automating complex interactions in testing.</div></div><ul><li><strong>Selenium WebDriver API</strong>: For web automation testing that requires simulating real user actions (like clicking, scrolling, and keyboard input), the Selenium WebDriver API provides a high-level interface to control the browser programmatically.&#xa0;</li></ul><div class="kg-card kg-callout-card kg-callout-card-blue"><div class="kg-callout-emoji">&#x1f44d;</div><div class="kg-callout-text"><b><strong style="white-space: pre-wrap;">Advantage</strong></b>: Beyond just selecting elements, Selenium allows for interaction with web elements in a way that closely mimics actual user behavior. This is particularly useful for end-to-end testing of web applications to ensure functionality, usability, and performance.</div></div><p>Each of these alternatives has its niche where it might outperform XPath, depending on the specific requirements of the task at hand. The choice between XPath and other methods depends on factors like the complexity of the web page, the nature of the dynamic content, the specific data or interactions required, and personal or project-based preferences for tooling and readability.</p><h2 id="conclusion">Conclusion</h2><p>With its&#xa0;<code>contains()</code>&#xa0;function, XPath is a great tool for web scraping and automation testing, as it enables extraction and interaction with dynamic web content. Understanding and applying XPath effectively can significantly enhance the efficiency and reliability of web scraping and automation tasks.</p>
