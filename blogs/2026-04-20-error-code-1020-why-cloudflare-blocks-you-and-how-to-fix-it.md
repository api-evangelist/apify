<img alt="Error code 1020: Why Cloudflare blocks you and how to fix it" src="https://storage.ghost.io/c/f2/6e/f26ec999-9a90-4aee-a0d4-9b3ca2bb668f/content/images/2025/02/error-code-1020.png" /><p>An error code 1020 means<a href="https://www.cloudflare.com/learning/what-is-cloudflare/" rel="noreferrer"> Cloudflare</a>&apos;s Security Firewall blocked your request because something about it violated firewall rules set by the website&apos;s owner.</p><p>The right solution depends on whether you&#x2019;re just trying to load a webpage or running a web scraping script.</p><p>This guide covers both scenarios.</p><figure class="kg-card kg-image-card"><img alt="Error code 1020: Why Cloudflare blocks you and how to fix it" class="kg-image" height="930" src="https://storage.ghost.io/c/f2/6e/f26ec999-9a90-4aee-a0d4-9b3ca2bb668f/content/images/2026/04/IMG_4965.webp" width="2000" /></figure><h2 id="what-is-error-code-1020">What is error code 1020?</h2><p>Every website that&#x2019;s protected by Cloudflare sits behind a security layer called a <a href="https://www.cloudflare.com/learning/ddos/glossary/web-application-firewall-waf/" rel="noreferrer">Web Application Firewall (WAF)</a>.</p><p>The site owner configures rules in this firewall that decide who gets in and who gets blocked. Error code 1020 is the response Cloudflare sends when your request fails one of those rules.</p><p>The reason could be a suspicious IP address, a browser fingerprint that doesn&apos;t match a real browser, HTTP headers that look automated, missing cookie data, or simply being in a country or region the site owner has chosen to block.</p><p>What makes this error different from a typical server error is that nothing is broken. The site is online, the server is responding, and Cloudflare itself is working exactly as intended.</p><p>The firewall simply evaluated your request and rejected it, based on the site owner&#x2019;s rules.</p><h2 id="what-causes-error-code-1020">What causes error code 1020?</h2><p>Some articles call 1020 a &#x2018;ChatGPT error,&#x2019; but that&#x2019;s just because ChatGPT is one of the sites that uses Cloudflare. Error 1020 is a firewall block that can happen on any website protected by Cloudflare, not something unique to ChatGPT. Here are the actual causes:</p><ol><li><strong>Poor IP reputation:</strong> VPNs and datacenter IPs are shared by thousands of users, including bots and scrapers. Cloudflare assigns many of these IPs low trust scores, which can trigger an instant block even on a first visit.</li><li><strong>Bot traffic behavior:</strong> Scripts, headless browsers, and crawlers that send requests without proper browser headers or TLS fingerprints are identified and blocked.</li><li><strong>Browser extensions altering request headers:</strong> Extensions that modify User-Agent strings, block scripts, or spoof headers can distort a browser&apos;s digital fingerprint enough to resemble bot traffic.</li><li><strong>Disabled cookies:</strong> Cloudflare uses clearance cookies to verify that a visitor has passed a security challenge. If your cookies are disabled, the WAF can&#x2019;t confirm that you&#x2019;re a legitimate visitor.</li><li><strong>Shared IP addresses:</strong> If another user on the same IP has triggered a block through abusive behavior, the entire IP can be flagged, affecting everyone sharing it.</li><li><strong>Geographic restrictions:</strong> Site owners can configure Cloudflare to block traffic from specific countries or regions entirely.</li><li><strong>Web scraping:</strong> Scraping tools that send high volumes of requests without rotating IPs, mimicking real browser behavior, or managing clearance cookies are a primary target for WAF rules.</li></ol><h2 id="how-to-fix-error-code-1020-for-everyday-users">How to fix error code 1020 (for everyday users)</h2><h3 id="1-turn-off-vpn-services">1. Turn off VPN services</h3><p>VPNs route your traffic through shared datacenter IP addresses, and because thousands of users (and bots) use those same IPs, they often carry poor trust scores on Cloudflare&apos;s network. A single bad actor on a shared VPN server can get the entire IP flagged, which means you inherit their reputation. Disabling the VPN reveals your actual residential IP, which usually has a cleaner reputation.</p><h3 id="2-restart-your-network-router">2. Restart your network router</h3><p>If you&#x2019;re not using a VPN and still getting blocked, your own IP address may be the issue. Restarting the router will prompt your Internet Service Provider (ISP) to assign you a new IP with a clean reputation. This doesn&#x2019;t always work because some ISPs use semi-static assignments that persist across reboots.</p><h3 id="3-make-sure-browser-cookies-are-enabled">3. Make sure browser cookies are enabled</h3><p>Cloudflare uses specific cookies to manage access. When you pass a security challenge, Cloudflare stores a cookie in your browser as proof that you&#x2019;ve been verified. If your browser has cookies disabled or a privacy extension is blocking them, Cloudflare can&#x2019;t verify your session, and the WAF is more likely to deny you access. Check your browser&apos;s privacy settings and make sure cookies are allowed for the site you&#x2019;re trying to visit.</p><h3 id="4-deactivate-browser-extensions-or-switch-browsers">4. Deactivate browser extensions or switch browsers</h3><p>Some browser extensions interfere with how your browser presents itself to websites, causing it to resemble illegal bots to Cloudflare. Try disabling all extensions and reloading the page. If the error disappears, turn extensions back on one at a time to find which one caused the block. Alternatively, open the same URL in a clean browser with no extensions installed, like a fresh Edge or Safari profile. If it loads, the problem is with your main browser&apos;s configuration.</p><h3 id="5-contact-the-site-administrator-with-your-ray-id">5. Contact the site administrator with your Ray ID</h3><p>If none of the above works, the block is likely caused by a specific firewall rule that only the site owner can change. Every 1020 error page includes a unique Ray ID at the bottom. This ID is a reference to the exact request that was blocked. Send it to the site administrator and ask them to look it up in their Cloudflare security log. From there, they can see exactly which rule flagged your request and decide whether to permit your IP or adjust the rule.</p><h2 id="how-to-bypass-cloudflare-error-1020-when-web-scraping">How to bypass Cloudflare error 1020 when web scraping</h2><p>If you&#x2019;re getting a 1020 while running scraping pipelines, the everyday user fixes above won&#x2019;t help because Cloudflare is blocking your scraper&apos;s identity itself. Instead, follow the techniques below to resolve the block (NB: These methods aren&apos;t standalone fixes; the most effective bypass techniques are a combination of foundational and advanced strategies):</p><h3 id="1-match-your-tls-fingerprint-to-a-real-browser">1. Match your TLS fingerprint to a real browser</h3><p>Before Cloudflare even looks at your HTTP headers, it analyzes the TLS handshake your client initiates. Using JA3/JA4 fingerprinting, it profiles your client based on cipher suites, extensions, and elliptic curves. </p><p>If that fingerprint doesn&apos;t match a real browser, the request gets blocked.</p><p>Standard libraries like Python&apos;s <code>requests</code>, Go&apos;s <code>net/http</code>, and Node&apos;s <code>axios</code> all produce TLS fingerprints that are easy to differentiate from real browsers. </p><p>Instead, use client libraries like <code>curl_cffi</code> (Python), <code>got-scraping</code> (Node.js), or <code>utls</code> (Go) that&#x2019;ll help you better match browser-like handshake behavior.</p><p>Here&apos;s how a request looks using Python&apos;s <code>curl_cffi</code> with a Chrome TLS fingerprint:</p><pre><code class="language-python">from curl_cffi import requests

response = requests.get(
    &quot;&lt;https://example.com&gt;&quot;,
    impersonate=&quot;chrome&quot;,
)
print(response.status_code)
</code></pre><p>And the equivalent using <code>got-scraping</code> in Node.js:</p><pre><code class="language-jsx">import { gotScraping } from &apos;got-scraping&apos;;

const response = await gotScraping({
    url: &apos;&lt;https://example.com&gt;&apos;,
    headerGeneratorOptions: {
        browsers: [&apos;chrome&apos;],
        operatingSystems: [&apos;windows&apos;],
    },
});
console.log(response.statusCode);
</code></pre><h3 id="2-replicate-the-full-http2-header-structure-of-a-real-browser">2. Replicate the full HTTP/2 header structure of a real browser</h3><p>Cloudflare also inspects HTTP/2 header frames, including pseudo headers like<code>:method</code>, <code>:authority</code>, <code>:scheme</code>, <code>:path</code>) and their ordering. If the structure matches a known HTTP library but the User Agent says Chrome, this mismatch can trigger a block.</p><p>Your headers must also be internally consistent. Chromium browsers send <code>Sec-Ch-Ua</code>, <code>Sec-Ch-Ua-Mobile</code>, and <code>Sec-Ch-Ua-Platform</code>. Firefox doesn&apos;t. Mixing Chromium Client Hints with a Firefox User Agent, or omitting headers the claimed browser would normally send, signals &#x201c;automation&#x201d;.</p><p>The safest approach is navigating to the Network tab in DevTools, copying the exact headers from a real browser request, and replicating them in your scraper exactly, including the order.</p><h3 id="3-route-traffic-through-residential-proxies">3. Route traffic through residential proxies</h3><p>Cloudflare assigns trust scores to IP addresses based on their origin. Traditional datacenter IPs from AWS or DigitalOcean get flagged instantly because real users don&#x2019;t browse from server farms. Rotating datacenter IPs will only trigger more 1020 blocks.</p><p>Residential proxies, however, route traffic through real IP addresses that belong to actual households. These generally carry higher trust scores on Cloudflare&#x2019;s network.</p><p>But Cloudflare keeps improving its detection defenses, targeting residential proxy networks by combining network-wide traffic data with client-side fingerprints collected from solved challenges across its network.</p><p>This means the quality of your proxies matters more than ever. Cheap, widely shared residential proxies are more likely to be detected, so invest in <a href="https://blog.apify.com/best-proxy-providers/" rel="noreferrer">reputable proxy providers</a> with large, diverse pools and low per IP usage density.</p><p>If you must use datacenter IPs, look for&#xa0;&quot;ISP Proxies&quot;&#xa0;(also called Static Residential). They are hosted in datacenters but use residential ASN tags to trick reputation systems</p><h3 id="4-use-stealth-modified-browser-automation">4. Use stealth-modified browser automation</h3><p>Browser automation is a valid bypass approach, but using Selenium, Playwright, or Puppeteer out of the box is one of the fastest ways to get detected. They leave signatures that Cloudflare can easily spot.</p><p>To use browser automation effectively, you need stealth tools like <a href="https://pypi.org/project/undetected-chromedriver/">undetected chromedriver</a>, <a href="https://pypi.org/project/nodriver/">Nodriver</a>, or <a href="https://pypi.org/project/camoufox/">Camoufox</a> to suppress WebDriver flags and mask your automation signatures.</p><p>Running in headed mode (with a visible browser window) also helps, because headless environments produce fingerprint differences that detection systems can identify.</p><h3 id="5-handle-cloudflare-turnstile-challenges">5. Handle Cloudflare Turnstile challenges</h3><p>Cloudflare replaced CAPTCHA challenges with Turnstile, a non-interactive challenge system that runs JavaScript in the background, analyzing browser environment, proof of work results, and interaction patterns.</p><p>Because Turnstile operates through JavaScript rather than visual puzzles, traditional CAPTCHA solving services won&#x2019;t apply here.</p><p>To handle Turnstile, you can use a&#xa0;real browser environment&#xa0;to execute the challenge natively,&#xa0;or&#xa0;a&#xa0;third-party API&#xa0;like <a href="https://www.capsolver.com/">CapSolver</a> to generate a token, provided&#xa0;your scraper also maintains a&#xa0;legitimate, consistent fingerprint&#xa0;to submit that token.</p><h3 id="6-maintain-cookies-and-session-state">6. Maintain cookies and session state</h3><p>Cloudflare uses <code>cf_clearance</code> to record that a visitor passed a challenge and <code>__cf_bm</code> as part of its bot management system. If your scraper discards cookies between requests, every single request is evaluated as a brand new visitor, forcing Cloudflare to verify you from scratch each time. This increases the chance of triggering a 1020.</p><p>Maintain a persistent cookie jar across all requests within a session. With rotating proxies, you need to bind each proxy IP to its own cookie jar so the cookies stay consistent with the IP address Cloudflare sees.</p><h3 id="7-validate-your-data-against-ai-labyrinth-honeypots">7. Validate your data against AI Labyrinth honeypots</h3><p>Bypassing a 1020 block doesn&#x2019;t guarantee you&#x2019;re extracting real data. Cloudflare&#x2019;s recent AI Labyrinth feature embeds hidden links on pages leading to AI-generated decoy content.</p><p>These links are invisible to human visitors but are designed to lure automated crawlers deeper into a maze of irrelevant pages, wasting your scraper&apos;s time and resources on useless data.</p><p>To detect if your scraper has been redirected into an AI labyrinth, monitor for content that is topically unrelated to the site you are targeting, URL paths that were not present in the site&apos;s original navigation, and sudden increases in extraction volume without a corresponding increase in meaningful data.</p><p>Next, build circuit breakers directly into your pipeline that enforce session depth limits, use strict regex lists to filter unexpected URL structure, and a small AI model to check if the text you&#x2019;re scraping actually matches your target topic.</p><h3 id="8-use-a-managed-scraping-platform">8.
