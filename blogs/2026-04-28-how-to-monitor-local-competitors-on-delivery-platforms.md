<img alt="How to monitor local competitors on delivery platforms" src="https://storage.ghost.io/c/f2/6e/f26ec999-9a90-4aee-a0d4-9b3ca2bb668f/content/images/2026/04/How-to-monitor-local-competitors-across-delivery-platforms-1.png" /><p>A restaurant chain can post solid brand-level numbers while individual locations are bleeding orders. A store whose delivery zone is filled with cheaper competitors gets masked by stronger sites elsewhere, and by the time monthly reports flag the dip, the platform is already routing customers somewhere else.</p><p>Manual spot checks don&#x2019;t scale past three or four locations. What you need is automated, zone-level visibility into how each store stacks up against the actual competitors a customer sees when they open DoorDash or Instacart in that ZIP code.</p><p>This tutorial walks through how to set that up using <a href="https://console.apify.com/actors/2APbAvDfNDOWXbkWf">E-commerce Scraping Tool</a> with n8n - pulling competitor listings alongside your own pages from DoorDash and Instacart, on a schedule, with structured output that drops straight into Google Sheets.</p><figure class="kg-card kg-image-card"><img alt="How to monitor local competitors on delivery platforms" class="kg-image" height="1092" src="https://storage.ghost.io/c/f2/6e/f26ec999-9a90-4aee-a0d4-9b3ca2bb668f/content/images/2026/04/workflow_diagram.png" width="2000" /></figure><h2 id="why-brand-level-delivery-data-isnt-enough">Why brand-level delivery data isn&apos;t enough</h2><p>Brand-wide averages warp location-level performance. A struggling store gets hidden by stronger sites, and the decline doesn&#x2019;t surface in dashboards until orders have already shifted to competitors.</p><p>Each location also operates in a different competitive set. Pricing that looks competitive in one city can be much higher than the local norms in another.</p><p>The data needed to fix this exists on DoorDash and Instacart, but it&#x2019;s unstructured, scattered across platforms, and impossible to aggregate at scale without automation.</p><div class="kg-card kg-callout-card kg-callout-card-blue"><div class="kg-callout-emoji">&#x1f4a1;</div><div class="kg-callout-text">Apify tools are called Actors. They can perform both simple actions - like filling out web forms or sending emails - and complex operations, such as crawling millions of web pages or transforming large datasets.</div></div><h2 id="how-e-commerce-scraping-tool-solves-this">How E-commerce Scraping Tool solves this</h2><p>E-commerce Scraping Tool is an Apify Actor that can pull product and pricing data from delivery platforms by location or ZIP code. Configure it with a location and a search keyword, and it returns a structured dataset of competitor listings. Schedule it to run daily, weekly, or monthly, and the result is a continuously updated, zone-level view of the market each of your locations is competing in.</p><h2 id="setting-up-zone-level-monitoring-with-apify">Setting up zone-level monitoring with Apify</h2><p>By the end of this tutorial, you&#x2019;ll have a running workflow that pulls competitor prices for every zone you care about and writes them into a single sheet, tagged by location and ready for comparison.</p><p>Let&#x2019;s begin.</p><div class="kg-card kg-callout-card kg-callout-card-blue"><div class="kg-callout-emoji">&#x2757;</div><div class="kg-callout-text">To follow along with this tutorial, make sure you have access to <a href="https://n8n.io/" rel="noopener noreferrer">n8n</a> and have both an active <a href="https://console.apify.com/sign-up" rel="noopener noreferrer">Apify account</a> and a Google account.</div></div><h3 id="step-1">Step 1:</h3><p>Open n8n, create a new workflow, and select a scheduled trigger as the first step. You can then set how often the workflow runs in the <strong>Trigger Rules.</strong></p><figure class="kg-card kg-image-card"><img alt="How to monitor local competitors on delivery platforms" class="kg-image" height="597" src="https://storage.ghost.io/c/f2/6e/f26ec999-9a90-4aee-a0d4-9b3ca2bb668f/content/images/2026/04/create_trigger.png" width="800" /></figure><figure class="kg-card kg-image-card"><img alt="How to monitor local competitors on delivery platforms" class="kg-image" height="1196" src="https://storage.ghost.io/c/f2/6e/f26ec999-9a90-4aee-a0d4-9b3ca2bb668f/content/images/2026/04/schedule_trigger.png" width="978" /></figure><h3 id="step-2">Step 2:</h3><p>To run the Actor on a loop, the input variables need to be an array of JSON objects. Each JSON needs to include both the search keyword, and the location/zip code. To limit credit usage, stick with two locations while testing - you can scale up later.</p><p>Create a JavaScript <strong>Code </strong>node and add you input in the following format. Then execute the step.</p><pre><code class="language-JSON">return [
  {json: {location: &quot;London&quot;, keyword: &quot;burger&quot;}},
  {json: {location: &quot;Manchester&quot;, keyword: &quot;burger&quot;}}
]</code></pre><figure class="kg-card kg-image-card"><img alt="How to monitor local competitors on delivery platforms" class="kg-image" height="746" src="https://storage.ghost.io/c/f2/6e/f26ec999-9a90-4aee-a0d4-9b3ca2bb668f/content/images/2026/04/create_code_block.png" width="1000" /></figure><figure class="kg-card kg-image-card"><img alt="How to monitor local competitors on delivery platforms" class="kg-image" height="866" src="https://storage.ghost.io/c/f2/6e/f26ec999-9a90-4aee-a0d4-9b3ca2bb668f/content/images/2026/04/initialize_input.png" width="954" /></figure><h3 id="step-3">Step 3:</h3><p>Next, create a loop so the Actor can scrape locations one at a time. Select the <strong>Loop Over Items (Split in Batches) </strong>operation as the next workflow step. Keep the batch size as one.</p><figure class="kg-card kg-image-card"><img alt="How to monitor local competitors on delivery platforms" class="kg-image" height="746" src="https://storage.ghost.io/c/f2/6e/f26ec999-9a90-4aee-a0d4-9b3ca2bb668f/content/images/2026/04/create_loop.png" width="1000" /></figure><h3 id="step-4">Step 4:</h3><p>Search for the Apify node in your canvas and install it (if you haven&apos;t already). Replace the placeholder node with the <strong>Run an Actor and get dataset </strong>operation.</p><figure class="kg-card kg-image-card"><img alt="How to monitor local competitors on delivery platforms" class="kg-image" height="746" src="https://storage.ghost.io/c/f2/6e/f26ec999-9a90-4aee-a0d4-9b3ca2bb668f/content/images/2026/04/create_actor_run.png" width="1000" /></figure><p>Next, connect n8n to your <a href="https://console.apify.com/sign-up" rel="noopener noreferrer">Apify account</a>. This can be done using OAuth2, or by pasting your Apify API key located in <a href="https://console.apify.com/" rel="noopener noreferrer">Apify Console</a> under Settings &#x2192; API &amp; Integrations. Once your Apify connection is established, select Apify&#x2019;s E-commerce Scraping Tool from the Actor list.</p><figure class="kg-card kg-image-card"><img alt="How to monitor local competitors on delivery platforms" class="kg-image" height="882" src="https://storage.ghost.io/c/f2/6e/f26ec999-9a90-4aee-a0d4-9b3ca2bb668f/content/images/2026/04/auth_actor_selection.png" width="962" /></figure><h3 id="step-5">Step 5:</h3><p>Initialize the Actor&apos;s input and map the location and keyword to the correct fields. The safest option is to paste the JSON below into the <strong>Input JSON </strong>field, and toggle from <strong>Fixed </strong>to <strong>Expression.</strong></p><pre><code class="language-JSON">{
    &quot;additionalProperties&quot;: false,
    &quot;additionalPropertiesSearchEngine&quot;: true,
    &quot;additionalReviewProperties&quot;: true,
    &quot;addressDelivery&quot;: &quot;{{ $json.location }}&quot;, 
    &quot;keywordDelivery&quot;: &quot;{{ $json.keyword }}&quot;,
    &quot;marketplacesDelivery&quot;: [
        &quot;www.instacart.com&quot;,
        &quot;www.doordash.com&quot;
    ],
    &quot;maxDeliveryResults&quot;: 10,
    &quot;maxProductResults&quot;: 10,
    &quot;scrapeInfluencerProducts&quot;: false,
    &quot;scrapeReviewsDelivery&quot;: false
}</code></pre><p>Since the Actor expects strings rather than n8n&apos;s default object passthrough, make sure that <code>{{ $json.location }}</code> and <code>{{ $json.keyword }}</code> are wrapped in quotation marks.</p><figure class="kg-card kg-image-card"><img alt="How to monitor local competitors on delivery platforms" class="kg-image" height="764" src="https://storage.ghost.io/c/f2/6e/f26ec999-9a90-4aee-a0d4-9b3ca2bb668f/content/images/2026/04/initialize_actor.jpg" width="800" /></figure><p>Run this step once so that n8n captures the Actor&apos;s output schema - later steps need to know what fields are coming back.</p><h3 id="step-6">Step 6:</h3><p>When the loop runs the Actor, the original <code>location</code> and <code>keyword</code> get separated from the Actor&apos;s output. This Set node stitches them back together so each row in the final sheet is tagged with the location it came from.</p><figure class="kg-card kg-image-card"><img alt="How to monitor local competitors on delivery platforms" class="kg-image" height="559" src="https://storage.ghost.io/c/f2/6e/f26ec999-9a90-4aee-a0d4-9b3ca2bb668f/content/images/2026/04/create_set_node.png" width="800" /></figure><p>Drag and drop the <code>location</code> and <code>keyword</code> items from the initial Code node into the input fields box. Make sure to <strong>Include Other Input Fields </strong>or else you will lose your Actor output when this step is run.</p><figure class="kg-card kg-image-card"><img alt="How to monitor local competitors on delivery platforms" class="kg-image" height="672" src="https://storage.ghost.io/c/f2/6e/f26ec999-9a90-4aee-a0d4-9b3ca2bb668f/content/images/2026/04/set_location_keyword.png" width="1000" /></figure><h3 id="step-7">Step 7:</h3><p>Connect a second Edit Fields (Set) node to the done branch of your Loop node. This will be used to filter out any unnecessary fields.</p><figure class="kg-card kg-image-card"><img alt="How to monitor local competitors on delivery platforms" class="kg-image" height="776" src="https://storage.ghost.io/c/f2/6e/f26ec999-9a90-4aee-a0d4-9b3ca2bb668f/content/images/2026/04/create_second_set_node.png" width="1644" /></figure><p>Drag and drop each item from the Edit Fields node. Some items will show as undefined until the workflow runs - that&#x2019;s expected.</p><figure class="kg-card kg-image-card"><img alt="How to monitor local competitors on delivery platforms" class="kg-image" height="761" src="https://storage.ghost.io/c/f2/6e/f26ec999-9a90-4aee-a0d4-9b3ca2bb668f/content/images/2026/04/map_loop_output_fields.png" width="800" /></figure><h3 id="step-8">Step 8:</h3><p>Create a Google sheet, name it, and add column headers that map to the fields from the previous step.</p><figure class="kg-card kg-image-card"><img alt="How to monitor local competitors on delivery platforms" class="kg-image" height="540" src="https://storage.ghost.io/c/f2/6e/f26ec999-9a90-4aee-a0d4-9b3ca2bb668f/content/images/2026/04/initialize_gsheet.png" width="1294" /></figure><p>Next, connect the Edit Fields node to an Append or update row in sheet operation. Then, connect your Google account.</p><figure class="kg-card kg-image-card"><img alt="How to monitor local competitors on delivery platforms" class="kg-image" height="838" src="https://storage.ghost.io/c/f2/6e/f26ec999-9a90-4aee-a0d4-9b3ca2bb668f/content/images/2026/04/create_gsheet_node.png" width="1554" /></figure><p>Select the sheet you created and map each JSON field to its column.</p><figure class="kg-card kg-image-card"><img alt="How to monitor local competitors on delivery platforms" class="kg-image" height="800" src="https://storage.ghost.io/c/f2/6e/f26ec999-9a90-4aee-a0d4-9b3ca2bb668f/content/images/2026/04/map_columns.png" width="674" /></figure><p>Run the workflow. The sheet should populate with one row per listing per location, tagged by city and keyword.</p><figure class="kg-card kg-image-card"><img alt="How to monitor local competitors on delivery platforms" class="kg-image" height="641" src="https://storage.ghost.io/c/f2/6e/f26ec999-9a90-4aee-a0d4-9b3ca2bb668f/content/images/2026/04/results.jpg" width="800" /></figure><h2 id="conclusion">Conclusion</h2><p>The setup above gives you a running, scheduled feed of what every customer in every zone actually sees when they open DoorDash or Instacart - your own listings alongside the competitor set, in one sheet, refreshed on whatever cadence you set. From there, the questions worth asking (&#x201d;which zones have we lost on price? Which have new entrants we hadn&#x2019;t noticed?&#x201d;) have data behind them.</p>
